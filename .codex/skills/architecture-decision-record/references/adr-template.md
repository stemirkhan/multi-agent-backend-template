# ADR Template

## ADR ID
`ADR-YYYYMMDD-<slug>`

## Status
`proposed | accepted | superseded`

## Context
- Current problem and boundaries.
- Constraints (latency, consistency, security, cost, operability).

## Decision
- Final architecture decision.
- Why this option is selected now.

## Alternatives Considered
1. `<option-1>`: pros / cons / risks
2. `<option-2>`: pros / cons / risks

## Consequences
- Positive outcomes.
- Negative outcomes and debt.

## Contract Impact
- API impact.
- DB impact.
- Worker/tests/monitor impact.

## Rollout Plan
1. `<owner>`: `<task>`
2. `<owner>`: `<task>`

## Rollback Plan
- Trigger condition.
- Safe rollback steps.

## Verification
- Checks/tests required.
- Expected verify result (`exit code 0`).

## Open Questions
- Items pending alignment.
